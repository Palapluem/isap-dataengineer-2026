# คำศัพท์ในงานนี้ อธิบายแบบสั้น

เอกสารและ code ใช้ชื่อเทคนิคภาษาอังกฤษเพื่อให้ตรงกับ SQL และการทำงานจริง แต่ความหมายของแต่ละคำไม่ควรเป็นอุปสรรคต่อการอ่านงาน

| คำ | ความหมายแบบง่าย | ในงานนี้ใช้ทำอะไร |
|---|---|---|
| `raw` | สำเนาข้อมูลต้นทางที่ยังไม่จัดรูปแบบ | เก็บชื่อไฟล์, sheet และ cell เพื่อย้อนกลับไปตรวจได้ |
| `staging` | พื้นที่เตรียมข้อมูลก่อนใช้งาน | แปลง Excel ที่อ่านยากให้เป็นตารางที่มีคอลัมน์ชัดเจน |
| `mart` | ตารางที่จัดไว้ให้คนวิเคราะห์ใช้ | เก็บข้อมูลกำลังพลและงบประมาณในรูปที่ query ได้สะดวก |
| `grain` | หนึ่งแถวแทนข้อมูลหนึ่งหน่วยอะไร | ช่วยกันไม่ให้รวมคนละประเภทหรือรวมยอดซ้ำ |
| fact table | ตารางตัวเลขที่ต้องการวิเคราะห์ | เช่น จำนวนคน, งบประมาณ และอัตราเบิกจ่าย |
| dimension | ตารางข้อมูลประกอบที่ใช้กรองหรือจัดกลุ่ม | เช่น หน่วยงาน, วันที่ และไฟล์ต้นทาง |
| lineage | เส้นทางที่บอกว่าข้อมูลมาจากไหน | ใช้ file hash, sheet และ row number เพื่อตามกลับไปที่ Excel |
| idempotency | รันไฟล์เดิมซ้ำแล้วผลลัพธ์ไม่บวม | ใช้ SHA-256 เพื่อ replace release เดิมแทนการเพิ่มข้อมูลต่อท้าย |
| data-quality check | กฎตรวจว่าข้อมูลอยู่ในสภาพที่รับได้ | ตรวจช่วงตัวเลข, ร้อยละ, ข้อมูลซ้ำ และยอดรวมที่เทียบได้ |
| manifest | บันทึกข้อมูล release ล่าสุดที่ระบบรู้จัก | ใช้เทียบว่าเว็บต้นทางมีไฟล์ใหม่หรือไม่ |
| `source_unavailable` | ระบบเข้าเว็บต้นทางไม่ได้ | แยกจาก `no_new_data` เพื่อไม่สรุปผิดว่าไม่มีไฟล์ใหม่ |

## ประโยคที่ใช้ในวันสัมภาษณ์

- แทนที่จะพูดว่า “เก็บ lineage” ให้พูดว่า “ทุกแถวย้อนกลับไปดูไฟล์, sheet และตำแหน่งใน Excel ได้”
- แทนที่จะพูดว่า “idempotent load” ให้พูดว่า “รันไฟล์เดิมซ้ำแล้วจำนวนข้อมูลไม่เพิ่ม”
- แทนที่จะพูดว่า “define grain” ให้พูดว่า “กำหนดก่อนว่า 1 แถวแทนอะไร เพื่อไม่รวมข้อมูลผิดความหมาย”
- แทนที่จะพูดว่า “source unavailable” ให้พูดว่า “เว็บเข้าไม่ได้ จึงยังบอกไม่ได้ว่าไม่มีข้อมูลใหม่”
